Triggering a {{ tto-name }} parent order at a specific price level

A {{tto-name}} order type supports triggers that let you manage when to start working the parent order, which will then start entering its child orders. One of the available ways is to set a trigger price, which puts the {{ tto-name }} parent order on hold until a trade occurs at a specific price level. When the market reaches that price level, the {{ tto-name }} parent order changes to Working state and submits its first child order. The {{tto-name}} order type supports the following types of triggers:

  • If-Touched: Triggers when a trade occurs at the specified price level or better
  • Stop: Triggers when a trade occurs at the specified price level or worse.

After selecting the type of trigger to use, you need to set a trigger price and trigger price type. The following trigger price types are supported:

  • LTP: Any trade occurs at the specified price level.
  • Bid/Ask: A trade on the selected side occurs at the specified price level.
  • Same/Opposite Side: A trade occurs at the specified price on the specified side of the market relative to the {{ tto-name }} parent order.

To set a trigger so the {{tto-name}} order will begin working when the market trades at a specific price on the same side as your Buy or Sell order, you would add a same-side If-Touched trigger as follows:

  1. Enable Trigger to set a trigger for the order.
  2. Select If Touched from the dropdown to trigger the {{tto-name}} order when a trade occurs at the specified price or better.
  3. For the Trigger price, enter the desired price at which to trigger the order.
  4. Select Same Side from the drop-down to indicate the trade must occur at the inside market for the same side as the TT Iceberg order. So if you enter the order as a Bid, the order will be triggered if a Buy order executes at the specified price.
