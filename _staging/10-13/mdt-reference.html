MD Trader Reference

Available settings

These settings affect only the selected MD Trader widget. To update the default settings with these value for newly-opened MD Trader widgets, or to expose an option to apply them to existing opened widgets, click Save as default.

Display

Display settings

Setting Description
Window Size Click to select the size of the widget.
Colors Allows you to customize or change the cell and column colors available in the widget.
Show price as net change Displays prices in the ladder as the difference between the last traded price and the settlement price for the instrument.
Highlight midpoint of last recenter Sets whether to show the midpoint (white line) after the ladder is re-centered.
Show aggregator leg markets If trading an aggregated instrument and you enable this setting, MD Trader will open the corresponding legs of the instrument in separate MD Trader widgets.
Tick interval per row Sets the multiplier for the tick size at each price in the ladder. For example, if an instrument tick size is .25 and you set Tick interval per row to "2", each price in the ladder would tick by .50. The tick interval can consolidate the display of prices in the price ladder and display a greater range of prices while still showing the total quantity available. If you click the checkbox to enable this setting, the Tick Scale field appears in the MD Trader widget with the tick interval value that you entered.
Auto recenter grid (seconds) Sets how many seconds before MD Trader automatically re-centers the price ladder.
Recenter when market has moved (ticks) Sets whether to automatically recenter the trading ladder when the market moves a specified number of ticks since the last time it was re-centered.
Show number of orders Shows/hides the BCnt and ACnt columns. Check this option to show both columns. Uncheck to hide the columns. The BCnt column shows the number of orders comprising the total bid quantity at a price level. The ACnt column shows the number of orders comprising the total ask quantity at a price level.
Only show direct prices When this setting is checked (enabled), implied prices are filtered out and only direct prices are shown, and implied bid and ask quantities are not included in the best bid and ask quantities. When unchecked (disabled), best bid and ask prices and quantities include both direct and implied values.
Show Liquidate button

Sets whether to show the Liquidate button that allows you to submit offsetting orders immediately to liquidate your position.

Note: This setting does not apply when MD Trader displays synthetic instruments, such as Autospreader or Aggregator instruments.
Show tabs Sets whether to display tabs at the bottom of the widget for the instruments used in the widget.
Reset order type to Limit after each order Sets whether to reset the Order Type drop-down to Limit automatically after an order with another order type is submitted.

Set Order type buttons

These settings allow you to add or remove buttons on the MD Trader widget for commonly used order types.

Button Description
+ Adds a new custom button
X Removes a new custom button
Order type Selects the order type to associate with the button
Label Sets the to display on the button
Preview Shows what the button will look like
Order Entry (local only)

The settings in the Order Entry (local only) section affect only the current MD Trader widget.

Order Entry (local only) settings

Setting Description
Maximum order quantity Sets the maximum quantity allowed per order.
2nd default order quantity Provides the ability to right-click in the Bids and Asks column at a price level to submit an order with a different default order quantity. To enable this feature, you must check the checkbox and type a quantity in the box to the right of this option.
Iceberg display quantity Sets the disclosed quantity to display when entering natively supported or TT Iceberg orders.
Stop Limit payup ticks from trigger Sets the maximum number of ticks away from the Stop trigger price to enter the Limit order.
Ignore global Account List broadcasts Configures the widget to ignore account selections made in an Account List. When checked (enabled), the widget ignores account selections broadcast by an Account List widget.
Save as default

The settings in this section apply only to the current widget. To change the default settings for all MD Trader widgets, click Save as default.

Context menu options

Right-click in the MD Trader widget outside of the price ladder to open the context menu. On this menu, you can select the following options:

  • Show/Hide Order Pane — Shows/hides the order pane, which contains all order entry options for the widget.
  • Show/Hide Header Pane — Shows/hides the header pane, which contains market data for the selected instrument in the widget.
  • Open — Opens additional widgets for the instrument:
    • Time & Sales — Open the Time & Sales widget for the selected instrument. For multiple instruments in MD Trader, click the tab for the instrument, first, then select Time & Sales from the context menu.
    • Chart — Opens a chart for the selected instrument. For multiple instruments in MD Trader, click the tab for the instrument, first, then select Chart from the context menu.
  • VAP — Exposes the VAP column to show volume-at-price data for each price level:
    • Show/Hide VAP — Shows or hides the column
    • Show bars — Shows or hides the VAP bars when VAP is shown
    • Split bid/ask volume — Displays the bar with blue and red to indicate the relative bid and ask quantities
    • Show values — Displays the total VAP as a number
    • Reset VAP — Resets the VAP counter for the widget and starts displaying VAP data that occurs after the reset. The VAP column header is hightlighted to indicate that the widget is displaying partial VAP data.
    • Restore VAP — Cancels the effect of a VAP reset and restores the display of VAP data to include historical values. Restoring VAP also clears the column heading highlighting.
  • Show/Hide PIQ — Shows or hides the PIQ column.
  • View my orders only: Whether to display the working orders only for the current user when multiple users share an account.

    Note: This setting does not affect the account position displayed in MD Trader.

  • Settings... — Opens the MD Trader settings menu.

Free-form text fields

The following table provides descriptions of the free-form text fields on the MD Trader order pane for exchanges supported by the TT platform.

Text Field Description Exchange API Mapping
Text A

May contain an optional, user-defined text value seeded from the Setup application that remains on the order in the TT system. This value may be routed to the exchange for clearing and back office purposes.

You can override the default value displayed on the Order Ticket or enter a new value if the field is blank. Alpha-numeric and special characters are allowed. The following exchanges have a maximum character length for this field:

  • ASX — 6 characters
  • BrokerTec — 32 characters
  • DGCX — 128 characters
  • EEX — 12 characters
  • Eurex — 12 characters
  • Euronext — 14 characters
  • ICE — 12 characters
  • ICE_L — 12 characters
  • MEFF — 15 characters
  • MX — 25 characters
  • SGX — 32 characters

The value in Text A is routed to the following exchanges (Setup app user exchange field is in parantheses):

  • ASX — Tag 58 (Text)
  • BrokerTec — user_id_s (Text)
  • DGCX — Tag 58 (Text A)
  • EEX — Tag 25007 (Text 1)
  • Eris — Tag 58 (Text)
  • eSpeed — Tag 58 (Text)
  • Eurex — Tag 25007 (Text 1)
  • ICE — Tag 9121 (Memo Field)
  • ICE_L — Tag 9121 (Memo Field)
  • KCG — Tag 58 (Text)
  • MEFF — Tag 58 (Text 1)
  • MX — Memo (Text 1)
  • OSE — customer_info_s (Text 1)
  • SGX — exchange_info_s (Text 2)
  • TFX — Tag 10001 (Text)
Text B

May contain an optional, user-defined text value seeded from the Setup application that remains on the order in the TT system. This value may be routed to the exchange for clearing and back office purposes.

You can override the default value displayed on the Order Ticket or enter a new value if the field is blank. Alpha-numeric and special characters are allowed. The following exchanges have a maximum character length for this field:

  • BrokerTec — 15 characters
  • EEX — 12 characters
  • Eurex — 12 characters
  • SGX — 15 characters

The value in Text B is routed to the following exchanges (Setup app user exchange property is in parantheses):

  • BrokerTec — exchange_info_s (Account)
  • EEX — Tag 25008 (Text 2)
  • Eurex — Tag 25009 (Text 3)
  • MEFF — Tag 1 (Instead of clearing account name)
  • OSE — exchange_info_s (Text 2)
  • SGX — customer_info_s (Text 1)
Text TT

Contains an optional, user-defined text value seeded from the Setup application that remains on submitted orders for tracking purposes in the TT system, but is not routed to the exchange. You can override the default value displayed on the Order Ticket or enter a new value if the field is blank.

None. This value is not sent to the exchange, but remains on the order within the Order Book, Audit Trail, and Fills widgets.
