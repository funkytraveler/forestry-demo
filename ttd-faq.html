  1. By default, TT Desktop installs to path: C:\Program Files\Trading Technologies\. Can it be moved to a different location?

    Yes. You can select a different default folder location while running the default installer.
  2. Can TT Desktop be delivered via SCCM?

    Yes. The default TTD executable file can be made into a Microsoft SCCM package to help manage deployments across multiple workstations. However, this method will not provide version control for the application.
  3. What updates do I need to manage for TT Desktop?

    TT Desktop is a local executable that connects to the services of the TT platform. This executable may require occasional updates. The TT platform services do not reside on the users machine so TT Desktop is always accessing the most recent version of the TT platform.
  4. When TT Desktop is launched, will it attempt to download any updates? Where will those updates be stored?

    Yes. By default, the latest version of the TT Desktop application is always downloaded when a user logs in. The application code is held in memory during the session, and is discarded when TT Desktop is closed or refreshed.
  5. Can I control the auto-update of the TT Desktop application?

    Yes. You can control the auto-update of the TT Desktop application by modifying the following Windows registry settings:
    • To disable auto-update for all users on the system, set the following registry key: `HKEY_LOCAL_MACHINE\Software\Trading Technologies\TT\WinSparkle\CheckForUpdates` = 0
    • To disable automatic updates for the current user, set the following registry key: `HKEY_CURRENT_USER\Software\Trading Technologies\TT\WinSparkle\CheckForUpdates` = 0
  6. Does the installation require an Internet connection?

    Yes. An Internet connection is required for the download of the TT Desktop installer but is not required to run the installation.
  7. Can I get a static version of TT software?

    No. A static version of TT Desktop is not available per the design of our software.
