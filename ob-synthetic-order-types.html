A synthetic order is an order managed within TT that submits orders to the exchanges. Synthetic orders provide functionality that is not natively supported by an exchange. Synthetic orders consist of a parent order and one or more child orders. When a trader submits a synthetic order, that order becomes the parent order; all native orders submitted to the exchange are child orders of that synthetic parent order. TT supports a variety of synthetic orders, including:

  • ADL® algos
  • Aggregator orders
  • Autospreader spreads
  • Staged orders
  • Strategies
  • TT order types

The Order Book displays synthetic orders with child orders nested below the parent order.

  1. Exchange-native order
  2. ADL Algo parent synthetic order
  3. TT Time Sliced synthetic parent order
  4. Autospreader parent order

The Order Book allows you to hide child orders by collapsing their parent orders.

The TTOrderID, ParentID, and Child Orders columns also provide information to identify parent synthetic orders and their associated child orders.

  • All parent and child orders have a TTOrderID.
  • Child orders also store the order ID for their parent orders in the ParentID column.
  • Parent synthetic orders indicate the number of associated child orders in the Child Orders column, while child orders always show 0 in that column.
