Interpreting order cross prevention messages

Order cross prevention messages appear in the Audit Trail and Fills widgets. Different messages are displayed based on the order cross prevention rule applied to your account.

Position Transfer messages

When the Position Transfer rule is applied to your account, orders submitted with any account in the account tree (on the branch at which Order Cross Prevention is set) are matched internally by the TT system, and Position Transfer messages are displayed in the Audit Trail and Fills widgets in the Trade application.

Example 1

For example, a 100 lot buy is resting at the exchange, and a 100 lot sell is submitted using the same account, which is configured with the Order Cross Prevention rule of “Position Transfer”. Based on the Position Transfer rule, the 100 lot buy and sell orders are matched internally at the resting order's Limit price and the original 100 lot buy is canceled at the exchange by the TT system. In the following Audit Trail, the Message column displays "Position Transfer" and the FillQty column shows the fully filled order quantity. Also, the message Reduced resting order <Exchange Order ID> for PosXfer is displayed to show the original order is canceled due to internal, pre-exchange matching for a full fill.

In the Fills widget for this example, the two internally matched Fills appear for the 100 lot buy and sell cross orders. The Route column shows "Internal," indicating the TT system canceled the working order and created fills for the 100 lot buy and sell orders to prevent order crossing at the exchange.

Example 2

In this example, a 100 lot buy order is resting at the exchange, and a 75 lot sell is submitted using the same account. Order Cross Prevention is set to Position Transfer for the account, so the 100 lot buy and 75 lot sell are matched internally at the resting order's Limit price. The 100 lot buy is partially filled for 75 contracts and the original quantity is replaced at the exchange with a new quantity of 25. In the following Audit Trail, the Message column displays "Position Transfer" to show the internally matched 75 lot sell and partially filled 100 lot buy with a new working quantity of 25, and the message Reduced resting order <Exchange Order ID> for PosXfer is displayed to show the original order quantity is replaced due to internal, pre-exchange matching for a partial fill.

In the Fills widget for this example, the two internally matched fills are logged: the 75 lot sell and the 100 lot buy partially filled for 75 contracts. The Route column shows "Internal," indicating the TT system replaced the original order and created a partial fill for the 100 lot buy and a full fill for the 75 lot sell orders to prevent order crossing at the exchange.

Cancel Resting (wait for ack) messages

When the Cancel Resting (wait for ack) order cross prevention rule is applied, the Canceled resting orders <Exchange Order ID> to avoid crossing orders message appears in the Audit Trail to indicate that the resting order has been canceled and that the new, potentially crossing order in the same account has been submitted.
